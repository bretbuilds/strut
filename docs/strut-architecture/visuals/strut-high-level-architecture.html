.claude/rules/ · architectural-decisions · system-map · .specs/archive
Knowledge substrate — next scan reads updated rules. Self-improving: scan detects gap → change built → update-capture proposes rule → human approves → next scan reads rule.
Modifier summary: Trust and decompose are independent binary switches set by classification. Trust ON adds: must_never collection in spec-derive-intent, negative criteria in spec-write, review-security (Opus) in review chain, impl-describe-flow before PR, mandatory knowledge capture with root-cause, and deeper human review at both gates. Decompose ON adds: task breakdown (≤5) in spec-write, decomposition validation in spec-review, per-task TDD iteration in run-implementation, and task 1 human gate. Guarded-decompose (both ON) adds: adversarial spec attack in separate session.
Architecture principles
Skills orchestrate (shared session context — dispatch + file-status checks, no reasoning work).
Agents do work (isolated context — fresh per dispatch, declared model via frontmatter, restricted tools via frontmatter).
Information crosses boundaries only through .pipeline/ file contracts. Orchestrators read only status fields to route.
Tests before implementation (structural — impl-write-code takes test files as input). Tests are permanent assets; code is disposable.
Anti-rationalization patterns added reactively: observe failure first, then add the directive.
