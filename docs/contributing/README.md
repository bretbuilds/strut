# docs/contributing/

Authoring tooling and templates for extending STRUT with new skills, agents, and specs. Use this folder when you're building out pipeline components — it is not adopter-facing integration material (that's in the main README).

## Subfolders

**`templates/`** — Blank templates for authoring skills, agents, and specs. Copy one to start a new component.

**`testing/`** — Dual-model test runner, assertion harness, and category guide for validating skills and agents. See `testing/README.md` for the workflow.

## Related

**`docs/strut-architecture/modifiers/`** — Cheat sheet and worked examples for the trust/decompose modifier system.
